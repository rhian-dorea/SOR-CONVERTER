# Sistema Universal de Conversão e Análise de Arquivos SOR

## Nome do Projeto

**SOR Converter Pro**

---

# 1. Objetivo

Desenvolver uma plataforma capaz de interpretar arquivos **.SOR (OTDR)** de diversos fabricantes, extrair todas as informações técnicas disponíveis e convertê-las para formatos amplamente utilizados, como PDF, Excel, CSV, JSON e XML.

Além da conversão, o sistema deverá oferecer análise dos dados, geração automática de relatórios e disponibilização das informações por meio de uma API REST.

---

# 2. Problema

Empresas de telecomunicações utilizam equipamentos OTDR para testar enlaces de fibra óptica.

Cada equipamento gera arquivos **.SOR**, que são binários e de difícil interpretação sem o software do fabricante.

Isso gera problemas como:

* Dependência do software do fabricante.
* Dificuldade em integrar dados com outros sistemas.
* Geração manual de relatórios.
* Pouca padronização.
* Baixa automação.

O sistema proposto eliminará essas limitações.

# 8. Módulos

## Upload

Responsável pelo envio dos arquivos.

Funções:

* Upload individual.
* Upload múltiplo.
* Drag and Drop.
* Validação.

---

## Detector

Identifica automaticamente:

* Fabricante.
* Versão.
* Estrutura.

---

## Parser

Cada fabricante possuirá seu próprio parser.

Exemplo:

```
parsers/

exfo.py
viavi.py
anritsu.py
yokogawa.py
veex.py
generic.py
```

Todos deverão gerar o mesmo modelo de saída.

---

## Modelo Padronizado

Todos os parsers devem produzir um objeto único contendo:

* Nome do arquivo
* Fabricante
* Modelo
* Número de Série
* Data
* Comprimento de onda
* Índice de refração
* Alcance
* Distância total
* Atenuação
* Perda Total
* ORL
* Eventos

---

# 9. Eventos

* Número
* Tipo
